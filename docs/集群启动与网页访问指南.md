# 大数据实训集群启动与网页访问指南

## 1. 服务器启动

1. **开机启动**：启动集群中的所有服务器
2. **登录服务器**：使用 root 用户登录主节点 hbase01-train

## 2. 基础服务启动

### 2.1 启动 Hadoop 集群

```bash
# 进入 Hadoop 安装目录
cd /export/servers/hadoop-3.2.2

# 启动 HDFS
./sbin/start-dfs.sh

# 启动 YARN
./sbin/start-yarn.sh

# 查看进程状态
jps
```

**预期结果**：
- hbase01-train 应看到 NameNode、ResourceManager 进程
- hbase02-train 应看到 SecondaryNameNode、DataNode、NodeManager 进程
- hbase03-train 应看到 DataNode、NodeManager 进程

### 2.2 启动 MySQL 服务

```bash
# 启动 MySQL 服务
systemctl start mysqld

# 验证 MySQL 服务状态
systemctl status mysqld
```

**预期结果**：MySQL 服务显示为 active (running)

### 2.3 启动 Tomcat 服务

```bash
# 进入 Tomcat 安装目录
cd /export/servers/tomcat-8.5.88

# 启动 Tomcat
./bin/startup.sh

# 查看 Tomcat 进程
ps aux | grep tomcat | grep -v grep
```

**预期结果**：Tomcat 进程正在运行

## 3. 服务状态验证

### 3.1 验证 Hadoop 状态

```bash
# 查看 HDFS 状态
hdfs dfsadmin -report

# 查看 YARN 节点状态
yarn node -list
```

**预期结果**：
- HDFS 显示 3 个 DataNode 在线
- YARN 显示 3 个 NodeManager 在线

### 3.2 验证 Hive 状态

```bash
# 启动 Hive metastore 服务
nohup hive --service metastore > /dev/null 2>&1 &

# 验证 Hive 连接
hive -e "show databases;"
```

**预期结果**：Hive 正常连接并显示数据库列表

## 5. 常见问题及解决方法

### 5.1 Hadoop 启动失败

**症状**：启动脚本执行后，部分进程未启动

**解决方法**：
1. 检查网络连接和主机名解析
2. 检查 Hadoop 配置文件
3. 查看日志文件：`/export/servers/hadoop-3.2.2/logs`

### 5.2 MySQL 启动失败

**症状**：MySQL 服务无法启动

**解决方法**：
1. 检查 MySQL 配置文件：`/etc/my.cnf`
2. 查看 MySQL 日志：`/var/log/mysqld.log`
3. 确保端口 3306 未被占用

### 5.3 Tomcat 启动失败

**症状**：Tomcat 服务无法启动或启动后无法访问

**解决方法**：
1. 检查 Tomcat 日志：`/export/servers/tomcat-8.5.88/logs/catalina.out`
2. 确保端口 8080 未被占用
3. 检查 Java 环境变量配置

### 5.4 网页访问出现 500 错误

**症状**：访问网页时显示 HTTP 500 错误

**解决方法**：
1. 查看 Tomcat 日志：`/export/servers/tomcat-8.5.88/logs/catalina.out`
2. 检查数据库连接配置
3. 确保相关服务（MySQL、Hive）已启动

## 6. 关闭服务

### 6.1 关闭 Tomcat

```bash
cd /export/servers/tomcat-8.5.88
./bin/shutdown.sh
```

### 6.2 关闭 Hadoop

```bash
cd /export/servers/hadoop-3.2.2
./sbin/stop-yarn.sh
./sbin/stop-dfs.sh
```

### 6.3 关闭 MySQL

```bash
systemctl stop mysqld
```

## 7. 注意事项

1. **服务启动顺序**：先启动 Hadoop，再启动 MySQL，最后启动 Tomcat
2. **服务依赖**：Hive 依赖 MySQL，Web 应用依赖 Hive 和 MySQL
3. **端口占用**：确保 8080（Tomcat）、3306（MySQL）、9000（HDFS）等端口未被占用
4. **防火墙**：确保防火墙允许相关端口的访问
